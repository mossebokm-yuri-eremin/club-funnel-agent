# SSH-ключи: настройка и деплой на Beget VPS

> Руководство для начинающих

---

## 1. Зачем нужен SSH-ключ?

SSH-ключ — это способ входа на удалённый сервер **без пароля**, но безопаснее, чем с паролем. Вместо того чтобы каждый раз вводить пароль, ваш компьютер «представляется» серверу с помощью цифровой подписи.

### Из чего состоит:

- **Приватный ключ** (`id_ed25519`) — хранится только у вас на компьютере. Никому не передавайте и не копируйте на сервер.
- **Публичный ключ** (`id_ed25519.pub`) — кладётся на сервер. Это «замок», к которому подходит только ваш приватный ключ.

### Где понадобится при деплое на Beget VPS:

- **Подключение к серверу через терминал** — вместо пароля каждый раз.
- **Git-деплой** — когда делаете `git pull` / `git clone` на сервере из приватного репозитория GitHub или GitLab, Git использует SSH-ключ для авторизации.
- **CI/CD-пайплайны** — автоматический деплой (GitHub Actions и др.) подключается к серверу по SSH-ключу без участия человека.
- **Загрузка файлов** — через `scp` или `rsync` прямо из терминала.
- **Настройка nginx, certbot, Docker** — всё это делается через SSH.

## 2. Создание SSH-ключа

Команды выполняются на вашем **локальном компьютере**, не на сервере.

### Windows (PowerShell)

Откройте PowerShell (`Win + X` → Windows PowerShell) и выполните:

```
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Программа спросит, куда сохранить ключ — нажмите **Enter** (путь по умолчанию). Затем предложит задать пароль — можно оставить пустым (Enter).

Ключи появятся в папке:

```
C:\Users\ВАШ_ПОЛЬЗОВАТЕЛЬ\.ssh\
```

Посмотреть публичный ключ:

```
cat ~\.ssh\id_ed25519.pub
```

### macOS (Terminal)

Откройте Terminal (`Cmd + Пробел` → Terminal) и выполните:

```
ssh-keygen -t ed25519 -C "your_email@example.com"
```

Так же нажмите Enter на вопросах о пути и пароле.

Ключи появятся в папке: `~/.ssh/`

Посмотреть публичный ключ:

```
cat ~/.ssh/id_ed25519.pub
```

### Как выглядит публичный ключ

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... your_email@example.com
```

Скопируйте **всю эту строку целиком** — она понадобится на следующем шаге.

## 3. Добавление ключа на Beget VPS

### Способ 1 — Через панель Beget (проще)

1. Зайдите в панель управления Beget → раздел **VPS**.
2. Найдите пункт **SSH-ключи** или **Авторизация**.
3. Нажмите «Добавить ключ» и вставьте содержимое файла `id_ed25519.pub`.
4. Сохраните.

### Способ 2 — Через терминал (универсальный)

Сначала войдите на сервер по паролю (первый и последний раз):

```
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

Затем выполните на сервере:

```bash
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "ВСТАВЬТЕ_СЮДА_СОДЕРЖИМОЕ_id_ed25519.pub" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

### Способ 3 — Одна команда (только macOS)

```
ssh-copy-id -i ~/.ssh/id_ed25519.pub root@IP_ВАШЕГО_СЕРВЕРА
```

---

## 4. Проверка подключения

После добавления ключа подключитесь к серверу:

```
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

Если всё настроено правильно — вы войдёте **без запроса пароля**. Терминал сразу покажет командную строку сервера.

## 5. Шпаргалка по командам

| Действие | Команда |
|---|---|
| Сгенерировать ключ | `ssh-keygen -t ed25519 -C "email"` |
| Показать публичный ключ (Windows) | `cat ~\.ssh\id_ed25519.pub` |
| Показать публичный ключ (macOS) | `cat ~/.ssh/id_ed25519.pub` |
| Подключиться к серверу | `ssh root@IP_АДРЕС` |
| Скопировать ключ на сервер (macOS) | `ssh-copy-id -i ~/.ssh/id_ed25519.pub root@IP` |

---

## Почему ed25519, а не rsa?

Более современный алгоритм: компактнее, быстрее, безопаснее. Все современные серверы, включая Beget, поддерживают его.
