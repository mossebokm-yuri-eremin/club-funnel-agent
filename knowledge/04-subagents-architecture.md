# Архитектура субагентов

## Концепция

**Аналогия из курса:**
- Ты — CEO компании
- Claude Code — генеральный директор
- Субагенты — отделы (специалисты по зонам)
- Skills — процедуры (стандартные операции)
- Knowledge base (CLAUDE.md + Context7) — источник знаний

Ты не нанимаешь каждого сотрудника лично. Говоришь Claude: "нужен специалист по X" — он находит, создаёт, интегрирует.

## Структура папки .claude/

```
.claude/
├── agents/

        └── SKILL.md             ← повторяемая операция
```

> CLAUDE.md — в корне проекта (не в .claude/)

## Стандартная команда субагентов

### database-architect (Opus 4.7)
```markdown
Зона: схема БД, миграции, RLS-политики, индексы, связи
НЕ трогает: фронт, API, бизнес-логику
Правила:
- Только через миграции (supabase/migrations/)
- RLS на каждой таблице
- snake_case для таблиц и колонок
- Миграции: YYYYMMDDHHMMSS_name.sql
- Деньги: integer × 100
- Никогда DELETE для пользовательских данных
```

### backend-engineer (Sonnet 4.6)
```markdown
Зона: API-роуты, бизнес-логика, серверные функции, интеграции
НЕ трогает: схему БД (только читает), UI-компоненты
Правила:
- TypeScript strict, запрещён any
- try/catch для всех async операций
- Входные данные — всегда валидировать (Zod)
- API ключи только из process.env
```

### frontend-developer (Sonnet 4.6)
```markdown
Зона: UI-компоненты, страницы, стилизация, клиентская логика
НЕ трогает: серверную логику, схему БД
Правила:
- Server Components по умолчанию
- 'use client' только для интерактивных компонентов
- Один компонент = один файл, максимум 200 строк
- Импорты через @/ алиас
- Tailwind v4 для стилей
- Error Boundaries для UI-ошибок
```

### qa-reviewer (Sonnet 4.6)
```markdown
Зона: ревью кода, тестирование, безопасность, соответствие спеке
НЕ правит код — только передаёт замечания другим агентам
Правила:
- Проверяет RLS на каждой таблице
- Проверяет отсутствие секретов в коде
- Проверяет Edge Cases из спеки
- Проверяет соответствие TypeScript типов схеме БД
```

## Как Claude генерирует команду субагентов

### Вход:
- SPEC.md (полная спецификация)
- CLAUDE_CODE_SETUP_GENERATOR.md (из Toolkits)
- Subagent Architect Guide (из Toolkits)

### Промт (в claude.ai, Plan Mode):
```
Прочитай SPEC.md, CLAUDE_CODE_SETUP_GENERATOR.md и Subagent Architect Guide.
Создай полный пакет настроек для Claude Code:
- CLAUDE.md (максимум 120 строк)
- .claude/agents/ (субагенты по спеке)
- .claude/rules/ (доменные правила)
- .claude/skills/ (повторяемые операции)
- MCP-команды для подключения Context7, Supabase, GitHub
- Финальный Kickoff промт для автономной сборки
```

## Модели для субагентов

| Субагент | Модель | Почему |
|----------|--------|--------|
| database-architect | **Opus 4.7** | Архитектурные решения, схема критична |
| backend-engineer | Sonnet 4.6 | Баланс качества/скорости |
| frontend-developer | Sonnet 4.6 | Рутинная сборка UI |
| qa-reviewer | Sonnet 4.6 | Достаточно для ревью |
| Аналитика данных | **Opus 4.7** | Сложный анализ |
| Категоризация | Haiku 4.5 | Быстро и дёшево |

> Opus для архитектуры и сложных решений. Sonnet для разработки. Haiku для простых повторяемых задач.

## Правила написания субагентов

**CLAUDE.md секция "Субагенты" (краткое описание):**
```markdown
## Субагенты
- database-architect (Opus) — схема БД, миграции, RLS, индексы
- backend-engineer (Sonnet) — API, бизнес-логика, интеграции
- frontend-developer (Sonnet) — UI-компоненты, страницы, стилизация
- qa-reviewer (Sonnet) — тестирование, ревью, безопасность
```

**Детали — в .claude/agents/[name].md:**
- Максимум 80 строк
- Зона ответственности (что делает)
- Зона запрета (что НЕ трогает)
- Правила работы
- Источники истины (какие файлы читает)
- Формат отчёта

## Skills (повторяемые операции)

```
.claude/skills/
├── create-migration/
│   └── SKILL.md    ← как создавать миграции
├── add-api-route/
│   └── SKILL.md    ← как добавлять API эндпоинт
└── add-component/
    └── SKILL.md    ← как создавать UI компонент
```

Skills — это инструкции для операций, которые повторяются много раз. Раз написал правильно → всегда одинаковое качество.
