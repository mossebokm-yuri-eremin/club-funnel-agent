# Библиотека типов агентов

> Читается при ARCHITECT: для определения рецепта, стека и ключевых вопросов.

---

## КАК ОПРЕДЕЛИТЬ ТИП

Первый вопрос в интервью: **"Что агент делает по сути — генерирует, хранит, анализирует, публикует или ведёт переговоры?"**

| Если главное действие... | Тип агента |
|--------------------------|------------|
| Генерирует и публикует контент | Контент-агент |
| Ведёт клиентов по этапам | CRM-агент |
| Отвечает в Telegram / отправляет сообщения | Бот-агент |
| Собирает и показывает данные | Аналитик-агент |
| Строит сайты и лендинги | Сайт-агент |
| Считает деньги и строит финмодели | Финансовый агент |
| Отслеживает участников сообщества | Комьюнити-агент |
| Подбирает поставщиков и ведёт сметы | Агент комплектации |
| Проектирует других агентов | Мета-агент (это мы) |

---

## ТИП 1 — КОНТЕНТ-АГЕНТ

**Что делает:** принимает тему/идею → генерирует пакет контента → публикует по расписанию

**Подтипы:**
- Telegram-контент (посты, опросы, анонсы)
- Instagram-контент (рилсы-сценарии, карусели, сторис)
- YouTube-контент (сценарии, описания, теги)
- Мультиплатформенный (один источник → все каналы)

**Обязательный стек:**
```
Claude API (Sonnet 4.6) — генерация текста
ChatPlace API — публикация в Telegram + автоматизации
Telegram Bot API — непосредственная отправка
Node.js + Supabase — хранение контент-плана и шаблонов
```

**Ключевые вопросы интервью:**
- На каких платформах публикуем? (Telegram / Instagram / YouTube / VK / все)
- Откуда идеи для контента? (Юрий наговаривает / расписание / триггер события)
- Нужна ли модерация перед публикацией или полный автопилот?
- Какой тон? (экспертный / личный / провокационный)
- Нужна ли мультиязычность? (RU + EN для международного рынка YE)

**Проекты-приоритеты:** YE, RZ

---

## ТИП 2 — CRM-АГЕНТ

**Что делает:** ведёт лиды по воронке от первого контакта до завершения

**Подтипы:**
- Продажи (лид → квалификация → переговоры → договор → деньги)
- Клиентский проект (договор → этапы → сдача → NPS)
- Наставничество (заявка → интервью → оплата → сессии → результат)

**Обязательный стек:**
```
Supabase (PostgreSQL + RLS) — хранение сделок и контактов
Telegram Bot API — уведомления менеджеру
Node.js — бизнес-логика воронки
ВАЖНО: данные клиентов → Beget VPS (152-ФЗ для РФ)
```

**Ключевые вопросы интервью:**
- Сколько этапов в воронке? (назови каждый)
- Кто пользуется: только ты или команда? (→ определяет права доступа)
- Нужна ли интеграция с Telegram-уведомлениями при каждом шаге?
- Есть ли финансовая составляющая (суммы сделок, авансы, остатки)?
- Нужна ли карточка клиента с историей всех коммуникаций?
- Интеграция с Google Sheets? (сметы, документы)

**Проекты-приоритеты:** VG, UA, YE (наставничество)

## ТИП 3 — БОТ-АГЕНТ (Telegram-воронка)

**Что делает:** отвечает на триггеры → ведёт пользователя по цепочке → конвертирует

**Подтипы:**
- Воронка-бот (кодовое слово → лид-магнит → прогрев → оплата)
- Поддержка (FAQ → ответы → эскалация на человека)
- Онбординг (новый участник → шаги → активация)
- Уведомительный (события → автосообщения)

**Обязательный стек:**
```
ChatPlace API — создание автоматизаций программно (150 кодовых слов!)
Telegram Bot API — отправка сообщений
Node.js — логика цепочек
Supabase — состояния пользователей
Beget VPS — деплой (webhook нужен публичный URL)
```

> ChatPlace ОБЯЗАТЕЛЬНО, не SaleBot — только у ChatPlace есть API для программного создания автоматизаций

**Ключевые вопросы интервью:**
- Что является триггером? (кодовое слово / подписка / кнопка / время)
- Какая цепочка после триггера? (опиши шаги)
- Нужен ли интерактив (кнопки, опросы, условная логика)?
- Что является целью конверсии? (оплата / регистрация / встреча)
- Нужна ли персонализация по сегментам аудитории?

**Проекты-приоритеты:** RZ (починить воронку!), YE (новая воронка)

## ТИП 4 — АНАЛИТИК-АГЕНТ

**Что делает:** собирает данные из разных источников → структурирует → выдаёт инсайты

**Подтипы:**
- Маркетинг-аналитик (Instagram, Telegram, YouTube, конверсии)
- Финансовый аналитик (доходы, расходы, P&L по проектам)
- Комьюнити-аналитик (активность, отток, сегменты участников)
- Бизнес-дашборд (все метрики в одном месте)

**Обязательный стек:**
```
Supabase — хранение метрик
Node.js + cron — периодический сбор
Claude API — интерпретация данных, инсайты
Telegram Bot — еженедельный отчёт Юрию
Опционально: Google Sheets MCP (если данные уже там)
```

**Ключевые вопросы интервью:**
- Какие решения будешь принимать на основе этих данных? (→ определяет метрики)
- Откуда данные? (API платформ / ручной ввод / Supabase)
- Как часто нужен отчёт? (ежедневно / еженедельно / по запросу)
- Нужен ли визуальный дашборд или текстовый отчёт в Telegram достаточно?
- Нужны ли алерты при отклонениях? (упал охват → уведомление)

**Проекты-приоритеты:** YE (платформы), RZ (клуб), VG/UA (P&L)

## ТИП 5 — САЙТ-АГЕНТ

**Что делает:** генерирует и публикует лендинги, обновляет контент, создаёт страницы

**Подтипы:**
- Лендинг под запуск (оффер + форма + оплата)
- Портфолио-агент (объект → страница с кейсом)
- Блог-агент (статья → SEO-оптимизация → публикация)

**Обязательный стек:**
```
Next.js 16 (App Router) + TypeScript + Tailwind v4 — веб
Vercel — деплой (если нет ПД) / Beget VPS (если есть ПД)
Supabase — CMS для контента
shadcn/ui — компоненты
UXUI Pro Max Skill (GitHub) — дизайн-компоненты
```

**Ключевые вопросы интервью:**
- Лендинг или многостраничный сайт?
- Нужна ли форма сбора заявок? (→ ПД → 152-ФЗ → Beget)
- Нужна ли оплата на сайте? (→ ЮKassa)
- Мобильная версия обязательна? (всегда да для RZ и YE аудитории)
- Нужен ли CMS (контент меняется часто) или статичный?

**Проекты-приоритеты:** VG (портфолио), UA (лендинг услуг), RZ (лендинг клуба)

## ТИП 6 — ФИНАНСОВЫЙ АГЕНТ

**Что делает:** принимает финансовые данные → структурирует → анализирует → рекомендует

**Подтипы:**
- Личный финдиректор (P&L по направлениям, фонды)
- Финмодель объекта (UA: отель, глэмпинг → ROI, Capex, Opex)
- Агент бюджетирования (план/факт по проектам)

**Обязательный стек:**
```
Telegram Bot API — интерфейс (загрузка выписок)
Supabase — хранение транзакций
Node.js — парсинг выписок (Альфа, Тинькофф, CSV)
Claude API — категоризация, анализ, инсайты
ВАЖНО: финансовые данные = ПД → Beget VPS
```

**Правило:** деньги всегда integer × 100 (копейки), никогда float

**Ключевые вопросы интервью:**
- Какие источники данных? (банк, крипта, Excel)
- Нужно ли разделение по проектам/направлениям?
- Нужны ли фонды? (налоговый, подушка, инвестиции)
- Нужна ли финмодель "что если" (прогнозы)?

**Проекты-приоритеты:** YE (личные финансы), UA (финмодели объектов)

## ТИП 7 — КОМЬЮНИТИ-АГЕНТ

**Что делает:** управляет жизнью сообщества — онбординг, удержание, активация

**Подтипы:**
- Онбординг-агент (новый участник → первые 7 дней → активация)
- Retention-агент (мониторинг → алерт при риске оттока)
- Геймификация-агент (баллы, достижения, рейтинги)
- Комьюнити-менеджер (автоответы, рекомендации, контент для чата)

**Обязательный стек:**
```
Telegram Bot API — все взаимодействия
ChatPlace — автоматизации цепочек
Supabase — профили участников, активность, сегменты
Node.js + cron — периодические проверки активности
Claude API — персонализированные сообщения
```

**Ключевые вопросы интервью:**
- Что значит "активный участник" в твоём клубе? (открывает уроки / пишет в чат / приходит на эфиры)
- Что предшествует уходу? (есть ли паттерн за 2 недели до отписки)
- Нужна ли персонализация? (участник A получает другое от участника B)
- Нужен ли куратор в петле или всё автоматически?
- Реферальная программа: как считать лиды амбассадора?

**Проекты-приоритеты:** RZ (100% отток — критично!)

## ТИП 8 — АГЕНТ КОМПЛЕКТАЦИИ

**Что делает:** ведёт спецификации проекта, подбирает поставщиков, контролирует сроки

**Подтипы:**
- Спецификатор (помещение → позиции → поставщики → цены)
- Логист (заказ → статус → доставка → монтаж)
- Сметчик (позиции → смета → PDF для клиента)

**Обязательный стек:**
```
Supabase — база позиций, поставщиков, статусов
Google Sheets MCP — если клиент работает в таблицах
Telegram Bot — уведомления о статусах
Node.js — бизнес-логика
```

**Ключевые вопросы интервью:**
- Работаете ли вы с фиксированным списком поставщиков или ищете каждый раз?
- Клиент видит спецификацию? (→ нужен экспорт PDF / веб-ссылка)
- Нужен ли контроль бюджета по позициям?
- Интеграция с проектом в CRM? (чтобы знать на каком этапе проект)

**Проекты-приоритеты:** VG, UA
