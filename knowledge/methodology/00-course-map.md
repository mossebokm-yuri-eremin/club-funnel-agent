# Карта курса — AI-Архитектор (ReveluxAI)

## Ключевая идея курса

**Ты — архитектор и коммуникатор. Claude — исполнитель.**
Не пишешь код вручную. Строишь правильную документацию → Claude собирает продукт автономно.

### Урок 3 — Установка инструментов разработки

**Что устанавливается:**
- Node.js LTS (nodejs.org)
- VS Code (code.visualstudio.com)
- Расширение Claude Code (Marketplace Anthropic)

**13 обязательных расширений VS Code:**

*Базовые (5 штук):*
- Tailwind CSS IntelliSense
- ESLint
- Prettier
- Git History
- GitLens

*Для БД (4 штуки):*
- Database Client / SQLTools
- Supabase (если есть)
- CSV Viewer
- Excel Viewer

*Для файлов (4 штуки):*
- PDF Viewer
- Office Viewer
- Markdown Preview
- ZIP Extractor

**4 режима работы Claude Code:**
| Режим | Что делает | Когда использовать |
|-------|------------|-------------------|
| Ask before edit | Спрашивает перед КАЖДЫМ изменением | Незнакомый проект |
| Auto-fix | Ключевые вопросы задаёт, мелкое — сам | Активная разработка |
| Plan Mode | Только планирует, НЕ делает | Создание CLAUDE.md, архитектура |
| Bypass Permissions | Полная автономия без подтверждений | Главный рабочий режим |

> Включить Bypass: Шестерёнка → Настройки → поиск "Bypass Permissions" → галочка → перезагрузить VS Code

### Урок 5 — Разработка спецификации (Spec-First Pipeline)

**Это центральный урок курса.**

**Последовательность:**
```
1. PROJECT_IDEA.md  (что мы строим)
        ↓
2. SPEC.md          (как именно строим — 500-1500 строк)
        ↓
3. CLAUDE.md        (бриф для каждой сессии, max 120 строк)
        ↓
4. Субагенты        (.claude/agents/*.md)
        ↓
5. Kickoff промт    → Claude Code → Автономная сборка
```

**Файлы из Toolkits курса:**
- `Project IDEA Template` — шаблон идеи
- `SPEC Template` — справочник структуры спецификации
- `SPEC Write Prompt` — промт для генерации SPEC.md
- `Agent Architect Prompt` — вставляется в Project Instructions claude.ai (не в чат!)
- `CLAUDE_CODE_SETUP_GENERATOR.md` — генерирует полный пакет (CLAUDE.md + субагенты + rules)
- `Subagent Architect Guide` — руководство по архитектуре субагентов

**Ключевые инсайты урока 5:**
1. Полчаса спеки = часы сэкономленной работы Claude Code
2. Описывай ВЕСЬ контекст бизнеса, даже если проект "не про бизнес"
3. Claude привирает со сроками — делить на 2
4. Opus 4.7 + Extended Thinking = обязательный комбо для спеки
5. Всегда добавляй блок "НЕ в MVP" и "НЕ наша аудитория"
6. Edge Cases — отдельный блок, минимум 10 сценариев

### Урок 8 — Подключение MCP

**MCP = Model Context Protocol** — "руки" для Claude.

| MCP | Установка | Зачем |
|-----|-----------|-------|
| Context7 | Глобально (один раз) | Актуальная документация всех библиотек |
| Supabase | Локально в проект | Работа с БД напрямую из Claude |
| GitHub | Локально в проект | Коммиты, PR, управление кодом |

> Context7 — глобально. Остальные — локально в каждый проект.

### Урок 9 — Supabase

**RLS (Row Level Security) — обязательно для каждой таблицы.**

4 типа операций: SELECT, INSERT, UPDATE, DELETE
- SELECT: пользователь видит только свои данные
- INSERT: пользователь может добавлять свои данные
- UPDATE: пользователь меняет только свои данные
- DELETE: **никогда не давать Claude право на DELETE пользовательских данных**

**Создание таблиц с MCP:**
- До MCP: 20-40 минут вручную
- После MCP: 1-2 минуты одним промтом

### Урок 20 — VPS на Beget (деплой)

**Зачем Beget, а не Vercel:**
- 152-ФЗ: персональные данные граждан РФ должны храниться в РФ
- Beget: дата-центр в СПб, проходит под закон
- ~11 ₽/день за VPS (1 ядро / 1GB / 10GB SSD)

**Порядок настройки VPS:**
1. Купить домен (Beget или Cloudflare)
2. Создать VPS (Ubuntu 24.04 LTS)
3. Создать SSH-ключ (ed25519)
4. Создать пользователя deploy (не root)
5. Отключить root по паролю
6. Настроить UFW firewall (22, 80, 443)
7. Установить Fail2Ban
8. Nginx + Certbot (SSL)
9. GitHub Actions (автодеплой)

## Архитектура субагентов

### Папка проекта:
```
.claude/
├── agents/         ← субагенты (специалисты)
├── rules/          ← правила домена
└── skills/         ← повторяемые операции
```

### Стандартная команда:
| Субагент | Модель | Зона ответственности |
|----------|--------|---------------------|
| database-architect | Opus 4.7 | Схема БД, миграции, RLS, индексы |
| backend-engineer | Sonnet 4.6 | API, бизнес-логика, интеграции |
| frontend-developer | Sonnet 4.6 | UI-компоненты, страницы, стили |
| qa-reviewer | Sonnet 4.6 | Тесты, ревью, безопасность |

> Opus для архитектурных решений. Sonnet для реализации. Не экономь на Opus — он окупит разницу.
