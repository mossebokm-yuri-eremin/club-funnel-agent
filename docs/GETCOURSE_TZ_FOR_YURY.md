# 🎯 GetCourse webhook → club-funnel-agent

**Цель:** настроить процесс в GetCourse, который при оплате клуба «Реализация»
шлёт webhook нашему агенту → агент фиксирует оплату и связывает её с воронкой
по `code_word` подписчика.

**Время:** 10 минут.
**Что нужно:** доступ админа в GetCourse.

---

## ✅ Чеклист — пройди по порядку

### 1. Войди в GetCourse как админ
Аккаунт `account.getcourse.ru`. Если не знаешь логин/пароль — в нашем
docs/SECRETS_INVENTORY.md.

### 2. Создай новый процесс
- `Настройки` → `Процессы`
- Кнопка `Создать процесс`
- Имя: **«Webhook на оплату клуба Реализация»**
- Тип объекта: **«Заказ»**
- Действие при триггере: **«Вызвать URL»**

### 3. Настрой триггер
В разделе **«Триггер процесса»**:
- **Тип события:** «Изменение статуса заказа»
- **Новый статус:** `Завершён успешно`
- **Дополнительное условие:**
  - Поле: «Предложение»
  - Оператор: «равно»
  - Значение: точное название продукта клуба
    *(зайди в `Магазин` → `Предложения`, скопируй название — например «Клуб
    Реализация», «Резидентство Реализация», или то, как оно у тебя названо)*

### 4. Настрой действие — вызов URL

В разделе **«Действие»** → `Вызвать URL`:

**URL (вставь как есть, одной строкой):**

```
https://agent.yury-eremin.ru/webhook/getcourse?event=payment_succeeded&order_id={object.id}&user_id={object.user.id}&user_email={object.user.email}&user_name={object.user.first_name}_{object.user.last_name}&user_phone={object.user.phone}&offer_id={object.offer.id}&offer_name={object.offer.title}&amount={object.cost_money}&paid_at={object.paid_at_time}&utm_source={object.user.utm_source}&utm_campaign={object.user.utm_campaign}
```

**Метод:** GET
**Заголовки:** не требуются.

### 5. Сохрани процесс
- Кнопка `Сохранить`
- Внизу страницы процесса включи тумблер **«Активен»** → должен загореться зелёным.

### 6. Тестовый запуск
- На странице процесса найди кнопку **«Тестировать»** (обычно справа сверху)
- Выбери любой завершённый заказ из списка
- Жми **«Запустить»**
- Запомни **точное время** теста по московскому

### 7. Сообщи мне время теста
Напиши боту:
```
Тест GC прошёл в 15:32 МСК
```
Я проверю в логах что webhook пришёл с правильными данными.

## 🚨 Возможные проблемы и решения

### ❌ «Нет поля {object.user.utm_source}»
Не все аккаунты имеют этот атрибут.
- Открой `Пользователь` → `Все поля` (любая карточка)
- Найди реальное имя UTM поля (обычно `extra.utm_source` или `field.utm_source`)
- Замени в URL: `utm_source={extra.utm_source}` или просто **удали этот параметр** из URL — не критично для теста.

### ❌ Предложение называется иначе
- `Магазин` → `Предложения`
- Найди продукт про клуб
- Используй его **точное название** в условии триггера (с учётом регистра и пробелов)

### ❌ Тест не уходит / webhook не доходит
1. Проверь что процесс **Активен** (зелёный тумблер)
2. В процессе → `История запусков` посмотри статус последнего теста
3. Если статус «Ошибка» — скопируй текст ошибки сюда

### ❌ Webhook доходит, но статус 4xx
- Возможно опечатка в URL (пропущенный `&` или `=`)
- Скопируй URL ещё раз из секции 4 — он там корректный

### ❌ Поля пустые в webhook
- Проверь что переменные доступны в типе процесса «По заказам»
- Если поле user.phone пустое — это нормально (не у всех клиентов телефон)
- Если order_id пустой — это ошибка типа процесса, проверь что выбран «Заказ»

## 📊 Что произойдёт после настройки

1. Клиент покупает клуб через GetCourse → платёж → статус заказа становится `Завершён`
2. GetCourse триггерит наш процесс → шлёт GET на `agent.yury-eremin.ru/webhook/getcourse?...`
3. Агент:
   - сохраняет raw event в `getcourse_raw_events`
   - парсер (cron 10 сек) нормализует payload
   - ищет `subscribers.email = {user_email}`
   - если есть → выставляет `subscribers.club_paid_at = NOW()`
   - связывает с funnel по code_word (если подписчик пришёл по нашей воронке)
   - шлёт тебе TG-уведомление: `💰 Оплата прошла! Иван П., email@x.ru — пришёл по коду klub_anna_350`
