# ТЗ для исполнителя — настройка GetCourse webhook «Оплата клуба Реализация»

**Кому:** оператор GetCourse / системный администратор
**Время:** 15-20 минут
**Сложность:** низкая (только UI GC, без кода)
**Кто принимает:** Юрий

---

## Контекст

Мы продаём подписку «Клуб Реализация» (5000 ₽/мес) в GetCourse. Когда клиент
оплачивает, GetCourse должен **сообщить** нашему серверу-агенту, чтобы агент
зафиксировал событие в БД, обновил аналитику воронки и отправил клиенту
приветствие через Telegram-бота.

Связь между GC и сервером-агентом — через **webhook (HTTP POST)**:

```
[Клиент оплатил оффер] → GC → POST https://agent.yury-eremin.ru/webhook/getcourse
                                     ↓
                                Сервер-агент: проверяет HMAC-подпись,
                                записывает в БД, шлёт TG-уведомление Юрию
```

Нужно настроить **один процесс** в админке GC, который шлёт этот POST.

---

## Доступы

| Что | Где |
|------|-----|
| Админка GC | `https://mossebo.getcourse.ru/admin` (логин + пароль возьмёшь у Юрия) |
| Оффер | «Бизнес клуб \| Доступ», ID `8313256` |
| URL агента | `https://agent.yury-eremin.ru/webhook/getcourse` |
| HMAC-секрет | возьмёшь у Юрия через защищённый канал (Bitwarden / личное сообщение) |

---

## Шаг 1 — Создать процесс

1. Зайти в **админку GetCourse**.
2. Меню → **«Задачи»** (или **«Процессы»**) → **«Создать процесс»**.
3. Заполнить:
   - **Название:** `Webhook: Оплата клуба Реализация`
   - **Объект:** `Заказы`
   - **Триггер:** `Изменение статуса`
4. Сохранить.

---

## Шаг 2 — Условия запуска

Открыть вкладку **«Процесс»** созданного процесса. Добавить условия:

| # | Поле | Оператор | Значение |
|---|------|----------|----------|
| 1 | Статус заказа | равно | `Завершён успешно` |
| 2 | Предложение в заказе | равно | `Бизнес клуб \| Доступ` (offer ID 8313256) |

Логика: оба условия должны выполняться (AND).

---

## Шаг 3 — Действие «Webhook»

Внутри того же процесса добавить **действие** → выбрать **«Webhook»** (или
**«HTTP-запрос»**). Заполнить:

| Поле | Значение |
|------|----------|
| **URL** | `https://agent.yury-eremin.ru/webhook/getcourse` |
| **Метод** | `POST` |
| **Content-Type** | `application/json` |

**Тело запроса (JSON)** — точно скопировать (имена переменных типа
`{order.id}` подставляются через кнопку «Вставить переменную» в GC):

```json
{
  "event": "purchase_completed",
  "order_id": "{order.id}",
  "user_id": "{user.id}",
  "user_email": "{user.email}",
  "user_name": "{user.first_name} {user.last_name}",
  "offer_id": "{order.offer.id}",
  "offer_name": "{order.offer.title}",
  "amount": "{order.amount}",
  "paid_at": "{order.paid_at}",
  "utm_source": "{user.utm_source}",
  "utm_campaign": "{user.utm_campaign}",
  "utm_content": "{user.utm_content}"
}
```

**Примечание:** если имена переменных типа `{order.id}` в твоей версии GC
отличаются — открой подсказку «Вставить переменную» и подбери эквиваленты
(GC сам предложит правильные имена).

---

## Шаг 4 — HMAC-подпись (защита от подделки)

В том же действии «Webhook» найти раздел **«Дополнительные заголовки»**
(или **«Заголовки»**). Добавить **один заголовок**:

```
X-GetCourse-Signature: HMAC-SHA256({body}, "СЕКРЕТ")
```

**Где `СЕКРЕТ`:** возьми у Юрия через защищённый канал (НЕ в открытом
чате/email). Это длинная hex-строка (64 символа).

GC сам вычислит HMAC-SHA256 от тела JSON с этим секретом и положит в
заголовок. Сервер-агент сделает то же самое и сравнит — если совпало,
запрос принят.

---

## Шаг 5 — Активировать и проверить

1. Сохранить процесс.
2. Нажать **«Активировать»** (иконка/кнопка в верхней панели процесса).
3. **Тестовая оплата:**
   - В GC включить «Режим тестовой оплаты» (если есть) — или попросить
     знакомого оплатить минимальную сумму через активный платёжный шлюз.
   - Оплатить оффер «Бизнес клуб \| Доступ».
4. **Проверить результат:**
   - В админке GC → процесс → вкладка **«Статистика»** → должна быть запись
     «Webhook отправлен → 200 OK».
   - У Юрия в Telegram **должно прийти уведомление от бота**: «Оплата клуба
     зафиксирована».

---

## Что вернуть Юрию

После завершения:

1. ✅ Скрин активированного процесса с надписью «Активен / Активирован».
2. ✅ Скрин из вкладки «Статистика» с записью «Webhook отправлен → 200 OK».
3. ✅ Скрин уведомления в Telegram от бота (если пришло).
4. Если что-то не работает — скрин ошибки + текст из логов GC процесса.

---

## Часто возникающие проблемы

| Симптом | Причина | Что делать |
|---------|---------|------------|
| `Webhook отправлен → 401 Unauthorized` | HMAC-секрет в GC ≠ секрет на сервере | Уточнить секрет у Юрия и пересохранить заголовок |
| `Webhook отправлен → 400 Bad Request` | JSON тело испорчено / отсутствует обязательное поле | Проверить тело JSON, особенно имена переменных `{order.id}` |
| `Webhook отправлен → 404 Not Found` | URL опечатан | Сверить URL: `https://agent.yury-eremin.ru/webhook/getcourse` (через `/webhook/getcourse`, не `/getcourse/webhook`) |
| `Webhook отправлен → 5xx` | Сервер-агент упал | Написать Юрию, он посмотрит логи |
| Webhook не отправляется вообще | Триггер не сработал — заказ не сменил статус | Проверить что заказ реально перешёл в «Завершён успешно» (не «В обработке») |

---

## Что НЕ нужно делать

- ❌ Не редактировать оффер «Бизнес клуб \| Доступ» — он уже настроен.
- ❌ Не менять статусы заказов вручную (триггер сработает на каждое изменение).
- ❌ Не публиковать секрет HMAC в чате/email/GitHub.
- ❌ Не настраивать ничего в ChatPlace — это делает агент автоматически.
